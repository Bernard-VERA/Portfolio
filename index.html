Projet

Création de la page d'accueil d'une agence de voyage


Description

Projet OpenClassrooms (Novembre 2023) :
Vous débutez une alternance en tant que développeur web au sein de la start-up Booki. L’entreprise souhaite développer un site Internet qui permette aux usagers de trouver des hébergements et des activités dans la ville de leur choix. Vous êtes chargé d'intégrer l'interface du site avec du code HTML et CSS. Cette interface devra être responsive et respecter la maquette fournie sur Figma.

Je sais maintenant comment intégrer une maquette, en partant de zéro. J'ai aussi découvert comment fonctionnent les Media Queries pour permettre à un site de s'afficher sur tout type d'écran. J'ai aussi appris à utiliser Git et GitHub pour versionner, sauvegarder et partager mon code.

Projet

Créer une page web dynamique avec JavaScript, pour le portfolio de Sophie Bluel.


Description

Projet OpenClassrooms (Décembre 2023):
Vous travaillez comme développeur front-end pour l’agence ArchiWebos qui comprend 50 salariés. Après la réunion pour échanger sur le projet, la cheffe de projet vous envoie en pièces jointes tout ce dont vous aurez besoin pour pouvoir réaliser vos missions, soit: le Figma du design qui vous servira à créer les nouvelles fonctionnalités, le code back-end qui vous permettra de faire persister les données et de tester les fonctionnalités, le code front-end d’origine avec la version statique de la page de l’architecte, et finalement, le Kanban pour la mission pour que vous puissiez voir les tâches qui vous sont attribuées. Vos 3 missions principales sont donc: développer la page de présentation des travaux de l'architecte (à partir du HTML fourni), développer la page de connexion de l'administrateur du site (le client) (code à créer de zéro), et développer la modale permettant d'uploader de nouveaux médias (code à créer from scratch).


Les problématiques

Les problématiques rencontrées sont:
- La récupération des données en passant par une API
- La gestion des évènements et la manipulation du DOM
- La création et la gestion de la modale. Tout cela en JavaScript.
- J'ai aussi dû construire une page de connexion pour l'administrateur, avec création et stockage d'un Token d'authentification.
j'ai pu résoudre tous ces problèmes grâce aux cours dispensés par Openclassrooms, mais aussi à mes recherches personnelles sur Internet.


Compétences

Cette expérience m'a permis d'acquérir des compétences solides en Javascript. J'ai appris à récupérer les données utilisateurs dans le JavaScript, via des formulaires et une API. Je sais gérer les évènements utilisateurs et manipuler les éléments du DOM. J'ai aussi découvert le fonctionnement d'un Kanban. J'ai trouvé ce projet trés difficile, mais mon envie d'apprendre a finalement pris le dessus. J'ai maintenant de bonnes bases en Javascript.

Cette compétence s'est avérée essentielle pour développer le projet et ajouter de nouvelles fonctionnalités complexes de manière efficace. Le Clerc d'Huissier

Projet

Un site d'information destiné aux employés des Huissiers


Description

Projet personnel (Mai 2009):
Avant d'être Développeur Web, j'ai eu une autre carrière, en tant que clerc d'huissier. Je suis fier de mon parcours dans le secteur juridique, mais j'ai toujours été attiré par l'informatique et Internet. Dés les années 2000, j'étais déja passionné par tout ce qui avait rapport au web. J'ai commencé à gérer des blogs, et à créer de petits sites familiaux (Les reseaux sociaux n'existaient pas encore). Je n'avais aucune formation dans ce domaine. Un jour, je me suis rendu compte qu'aucun site n'existait sur les clercs d'huissiers. J'ai donc décidé d'en créer un moi-même. Et je dois avouer que ce site à eu du succés. J'ai été référencé dans les 5 premiers de la recherche du mot "Huissiers" à une époque. Bien sûr, il est maintenant obsolète et démodé. Mon prochain projet sera sans doute d'en créer un nouveau, plus attrayant, sécurisé, et adapté aux smartphones, grâce à toutes les compétences que j'ai pu acquérir durant ma formation de Développeur Web, chez OpenClassrooms


Les problématiques

Mes compétences étaient trés limitées en 2009. J'ai donc utilisé un CMS gratuit (qui n'existe plus aujourd'hui). Je l'avais modifié pour qu'il corresponde à ce dont j'avais besoin. Je faisais des mises à jour régulières du site et de son contenu. Chaque opération me semblait compliquée, mais je suis toujours arrivé à obtenir le résultat attendu. Mes recherches sur Internet étaient ma seule solution pour régler ces problèmes.


Compétences

Cette expérience m'a permis d'apprendre à créer, mettre en ligne, et maintenir un site web. J'avais la satisfaction de voir que le forum du site aidait parfois les employés d'huissiers à trouver la solution à leurs problèmes. En 2009, Internet était encore en "bas débit", les tablettes et les smartphones n'existaient pas. Les interfaces étaient plutôt austères et le "https" n'existait pas. J'avais aussi créé quelques blogs sur Skyblog (qui n'existe plus maintenant). L'arrivée des réseaux sociaux à révolutionné Internet.

Clerc de commissaire de justice

Projet

Un site d'information destiné aux employés des Huissiers


Description

Projet personnel (Juillet 2024) :
Il s'agit de la refonte complète de l'ancien site "Le clerc d'huissier".


Les problématiques

L'ancien site n'était plus d'actualité. Il s'agissait d'un ancien CMS, inadapté aux formats tablette et mobile, et qui ne répondait plus au normes de sécurité de base. La mise à jour de l'ancien site était devenue impossible. Donc la seule solution était d'en créer un nouveau. De plus, une réforme de 2020 à changé profondément la profession. La convention collective aussi à été totalement remaniée.


Compétences

Ce projet me permet de vérifier et d'améliorer les compétences acquises lors de ma formation. Je suis parti de zéro. J'ai utilisé React pour créer cette "Single Page Application". Pour plus de fluidité, pas de base de données. J'ai créé des fichiers JSON pour stocker les textes. Des fonctions Javascript permettent de récupérer images et textes, et de les afficher dans des composants réutilisables. J'ai travaillé dans un repository Github et le site est hébergé gratuitement sur Vercel. Le site est adapté aux formats Dektop, Tablette et Mobile ( Responsive Design ). Contrôle du score Lighthouse, vérification de l'accessibilité avec Wave, SEO, vérification des erreurs avec les Chrome DevTools, liens vers d'autres sites. Tout a été vérifié. Ce site semble fonctionnel. Cet exercice m'a permis de voir les limites d'une "Single page application", notament au niveau du référencement : Une seule page, donc un référencement réduit.
